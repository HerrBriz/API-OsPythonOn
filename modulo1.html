Módulo 1

Scrum

A metodologia Scrum é um modelo ágil de gerenciamento voltado para desenvolvimento de software, mas que também pode ser aplicada em outras áreas. Ele se baseia em fornecer entregas contínuas e incrementais dos produtos, além de se adaptar as mudanças e incentivar o trabalho em equipe. Dentro desta metodologia a formação da equipe é baseada em três papéis fundamentais:

• Product Owner (P.O): É o indivíduo responsável por representar os interesses do cliente, elaborar o backlog do produto e garantir que a equipe foque nos itens de maior valor.
• Scrum Master: É o indivíduo responsável por gerenciar o Scrum Team, garantir que a equipe esteja em conformidade com a metodologia Scrum, além de eliminar os possíveis impedimentos que surjam.
• Scrum Team: É a equipe composta pelos membros que realizarão o desenvolvimento do software.

Durante a realização do Scrum, alguns eventos são de suma importância para garantir o andamento do projeto. São eles:

• Sprint: É a entrega contínua e adaptada do Mínimo Produto Viável (MVP).
• Daily Scrum: Refere-se a reunião diária e rápida do grupo para alinhar as atividades, identificar impedimentos e ajustar o plano de trabalho.
• Sprint Review: Refere-se à inspeção da Sprint e determinar futuras adaptações.

Desenvolver projetos baseado no método Scrum, pode trazer inúmeros benefícios para os desenvolvedores. São eles:

• Flexibilidade e Adaptabilidade: O Scrum permite ajustar e adaptar requisitos de forma ágil, respondendo rapidamente ao ambiente de negócios e as necessidades dos stakeholders.
• Colaboração e Comunicação: As Daily Scrum e feedback contínuo, permite a comunicação entre a equipe.
• Responsabilidade e Transparência: As equipes no Scrum são autogerenciáveis, permitindo que as responsabilidades sejam delegadas para os integrantes da equipe, enquanto a transparência no progresso de desenvolvimento aumenta a confiança entre todos os envolvidos.

Artefatos Essenciais - SCRUM

Nenhum cliente deseja um produto genérico, sempre precisam de um diferencial no que estão vendendo, por isso, aspectos únicos são necessários, com os artefatos SCRUM isso é facilitado.

Definição

Artefatos SCRUM servem para detalhar o produto, uma visão clara sobre cada detalhe de sua entrega é primordial para um bom desenvolvimento, ter isso documentado ajudará na organização da equipe.

Artefatos Essenciais

Sprint

Sprints são entregas fracionadas. Em um projeto existem diversas Sprints com objetivos diferentes, melhorando cada vez mais até chegar na Sprint final. Uma Sprint serve para o comprador ter noção do andamento de seu produto, verificando se tudo está indo conforme o planejado.

Backlog do produto

O Backlog é uma lista ordenada, indo da tarefa mais importante para a menos importante (quem define o nível de importância é o PO junto com o Dev Team). Backlog da Sprint são os requisitos que o cliente colocou, mas com ordem de prioridades, conforme o projeto for avançando, será necessário preenchê-lo com as novas ideias dele. Na primeira entrevista com o cliente é necessário apresentar o Backlog do produto para ele, com o intuito de confirmar que ambos estão com a mesma compreensão.

Backlog da Sprint

Lista ordenada com cada demanda distribuída nas Sprints, é uma estimativa do que a equipe é capaz de entregar em cada Sprint. Na primeira Sprint deve-se priorizar entregar algo funcional, mesmo que não esteja bonito, o cliente conseguirá fazer algo com aquele protótipo.

Definition of Done (DoD) e Definition of Ready (DoR)

Definition of Done é o valor mínimo que uma entrega precisa ter para ser considerada feita, por exemplo, se alguém mandar um arquivo sem título, sendo que no DoD específica que precisa de um, então essa tarefa ainda não foi finalizada. Definition of Ready serve para checar se o time possui tudo que é preciso para começar a trabalhar em certos pontos, como ter um conhecimento x para realizar a atividade y.

Exemplos:

Receitas são tutoriais para criar algo, cada receita é diferente da outra, mesmo tentando atingir o mesmo objetivo, como um bolo de laranja, por exemplo (um pode colocar mais farinha, outro mais ovos, etc). Então cada receita é um produto diferente e cada ingrediente é um ARTEFATO SCRUM.
Imagine várias confeitarias diferentes, se cada uma vender o mesmo bolo de laranja, ela não irá se destacar das outras, aquela que possuir os melhores ingredientes ou os mais criativos (que combinam melhor para formar um sabor mais agradável) conseguirá vender mais.
LEMBRE-SE: O gosto de cada um é exclusivo, então tente fazer o bolo de laranja que mais agrade o seu cliente (Escolha os ingredientes que ele gosta/precise).
